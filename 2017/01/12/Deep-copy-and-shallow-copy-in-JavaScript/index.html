JavaScript 中的深复制和浅复制

发表于 2017-01-12   |   分类于 学习笔记   |     |   阅读次数

对于 JavaScript 中的 Object 和 Array 类型，存在两种复制方式：深复制和浅复制。

直接赋值

显然直接赋值只是换了个变量名，指向的是同一个对象，这种复制方法是没有意义的。

浅复制

可以看到，浅复制只复制了一层对象属性，当对象里嵌套了子对象或者子数组时，只复制了子对象或子数组的引用。

深复制

1. 通过 JSON 对象的 parse 和 stringify 方法

上述方法可以很容易的实现对象的深复制，但是对于不支持JSON对象的数据类型（如 Function 和 Date 等）无法实现深复制。

2. 实现最简单的深复制

深复制对原始对象做了完整的复制，用递归的方法复制了其所有子对象。对深复制的对象进行修改不会影响原对象的值。

当对象很大，层级很多时，深复制会产生性能上的问题。
